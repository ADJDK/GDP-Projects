## The Story Behind This 

Hi! My name is Adrik Joseph. When I started using GitHub, I wanted a way to preserve my historical coding timeline. However, downloading my 3-year-old OnlineGDB files locally reset their creation dates to "today." 

Instead of manually editing 100+ files, I treated this data migration as an engineering challenge:
*   **Metadata Extraction:** I learned **JavaScript** to write a console web-scraper that pulled my original 2023 timestamps and project names directly from the OnlineGDB backend tables into a clean JSON mapping.
*   **Pipeline Automation:** I built a **Python pipeline script** to automate unzipping the files and handle mismatched names by dynamically stripping out symbols (`-` and `_`) for a clean folder match.
*   **Version Control Engineering:** I worked through terminal configurations with AI guidance to tap into Git's internal parameters (`GIT_AUTHOR_DATE` and `GIT_COMMITTER_DATE`), safely overriding local OS limitations to lock in my authentic learning history.

Every backend has its loopholes. By understanding Git's chronological logging engine, I fixed my tracking problem and applied this trick responsibly to protect my genuine developer history.
